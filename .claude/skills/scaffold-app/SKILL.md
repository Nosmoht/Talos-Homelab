# Scaffold Application

Use this skill to create the full directory structure and baseline manifests for a new application in this homelab GitOps repository. The generated files follow established conventions (forgejo, dex, etc.) and are immediately ready for ArgoCD sync.

## Inputs

- **app-name** (required): kebab-case name for the application (e.g., `gitea`, `vaultwarden`).
- **--project** (optional): `apps` (default) or `infrastructure`. Determines sync-wave, namespace prefix, and directory placement.
- **--helm** (optional): `<repo> <chart> <version>` — when provided, the Application CR uses a Helm source instead of the Git repo path. Not generated by default.

## Step 1: Validate Inputs

1. Parse the argument string for `app-name`, `--project`, and `--helm`.
2. Derive variables:
   - `PROJECT` = `apps` or `infrastructure`
   - `APP_DIR` = `kubernetes/overlays/homelab/${PROJECT}/${APP_NAME}`
   - `NAMESPACE` = `${PROJECT}-${APP_NAME}` (e.g., `apps-vaultwarden`)
   - `SYNC_WAVE` = `"3"` for apps, `"0"` for infrastructure
3. Verify the target directory does not already exist:
   ```bash
   ls kubernetes/overlays/homelab/${PROJECT}/${APP_NAME} 2>/dev/null && echo "ALREADY EXISTS"
   ```
   If it exists, stop and warn the user.

## Step 2: Create Directory Structure

Create the app directory and resources subdirectory:

```bash
mkdir -p kubernetes/overlays/homelab/${PROJECT}/${APP_NAME}/resources
```

## Step 3: Generate Manifests

### 3a. `${APP_DIR}/kustomization.yaml`

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
  - application.yaml
```

### 3b. `${APP_DIR}/application.yaml`

**Git source (default):**

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: ${APP_NAME}
  namespace: argocd
  annotations:
    argocd.argoproj.io/sync-wave: "${SYNC_WAVE}"
  labels:
    app.kubernetes.io/name: ${APP_NAME}
    app.kubernetes.io/instance: homelab
    app.kubernetes.io/component: application
    app.kubernetes.io/part-of: homelab
    app.kubernetes.io/managed-by: argocd
spec:
  project: ${PROJECT}
  source:
    repoURL: https://github.com/Nosmoht/Talos-Homelab.git
    targetRevision: main
    path: kubernetes/overlays/homelab/${PROJECT}/${APP_NAME}/resources
  destination:
    server: https://kubernetes.default.svc
    namespace: ${NAMESPACE}
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    retry:
      limit: 5
      backoff:
        duration: 5s
        factor: 2
        maxDuration: 3m
    syncOptions:
      - ServerSideApply=true
      - SkipDryRunOnMissingResource=true
```

**Helm source (when `--helm` provided):**

Replace `spec.source` with:

```yaml
  source:
    repoURL: ${HELM_REPO}
    targetRevision: ${HELM_VERSION}
    chart: ${HELM_CHART}
    helm:
      releaseName: ${APP_NAME}
      valuesObject: {}
```

And remove the `path` field.

### 3c. `${APP_DIR}/resources/kustomization.yaml`

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
  - namespace.yaml
  - cilium-networkpolicy.yaml
```

### 3d. `${APP_DIR}/resources/namespace.yaml`

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: ${NAMESPACE}
  labels:
    app.kubernetes.io/name: ${APP_NAME}
    app.kubernetes.io/instance: homelab
    app.kubernetes.io/component: namespace
    app.kubernetes.io/part-of: homelab
    app.kubernetes.io/managed-by: argocd
    pod-security.kubernetes.io/enforce: baseline
    pod-security.kubernetes.io/audit: restricted
    pod-security.kubernetes.io/warn: restricted
    platform.io/network-interface-version: v1
    platform.io/network-profile: restricted
```

### 3e. `${APP_DIR}/resources/cilium-networkpolicy.yaml`

Baseline CNP: DNS egress, kube-apiserver egress on port 6443, ingress from `ingress` entity (not `world`).

```yaml
apiVersion: cilium.io/v2
kind: CiliumNetworkPolicy
metadata:
  name: ${APP_NAME}
  namespace: ${NAMESPACE}
  labels:
    app.kubernetes.io/name: ${APP_NAME}
    app.kubernetes.io/instance: homelab
    app.kubernetes.io/component: network-policy
    app.kubernetes.io/part-of: homelab
    app.kubernetes.io/managed-by: argocd
spec:
  endpointSelector:
    matchLabels:
      app.kubernetes.io/name: ${APP_NAME}
  ingress:
    - fromEntities:
        - ingress
      toPorts:
        - ports:
            - port: "8080"
              protocol: TCP
  egress:
    - toEndpoints:
        - matchLabels:
            k8s:io.kubernetes.pod.namespace: kube-system
            k8s-app: kube-dns
      toPorts:
        - ports:
            - port: "53"
              protocol: UDP
            - port: "53"
              protocol: TCP
          rules:
            dns:
              - matchPattern: "*"
    - toEntities:
        - kube-apiserver
      toPorts:
        - ports:
            - port: "6443"
              protocol: TCP
```

**Note:** The ingress port `8080` is a placeholder. Inform the user to adjust it to the actual application port.

## Step 4: Update Parent Kustomization

Read `kubernetes/overlays/homelab/${PROJECT}/kustomization.yaml` and append the new app directory to its `resources` list. Maintain alphabetical order.

```bash
# Example: add the entry
yq -i '.resources += ["${APP_NAME}"]' kubernetes/overlays/homelab/${PROJECT}/kustomization.yaml
# Then sort
yq -i '.resources |= sort' kubernetes/overlays/homelab/${PROJECT}/kustomization.yaml
```

Verify the entry was added:

```bash
grep "${APP_NAME}" kubernetes/overlays/homelab/${PROJECT}/kustomization.yaml
```

## Step 5: Validate

Run `kustomize build` on the app directory to verify the generated manifests are valid:

```bash
kustomize build kubernetes/overlays/homelab/${PROJECT}/${APP_NAME}
```

If validation fails, fix the issue before proceeding.

Print the following checklist for the user:

```
  - ${APP_DIR}/application.yaml
  - ${APP_DIR}/resources/kustomization.yaml
  - ${APP_DIR}/resources/namespace.yaml
  - ${APP_DIR}/resources/cilium-networkpolicy.yaml

Updated:
  - kubernetes/overlays/homelab/${PROJECT}/kustomization.yaml

Post-scaffold TODO (handle conversationally, not auto-generated):
  [ ] Adjust CNP ingress port from placeholder 8080 to actual app port
  [ ] Add capability labels to namespace if consuming platform services
      (e.g., platform.io/consume.monitoring-scrape: "true")
  [ ] Verify AppProject "${PROJECT}" permits the resource kinds you will add
      (check kubernetes/overlays/homelab/infrastructure/argocd/resources/)
  [ ] Add SOPS secrets if needed (ksops-generator.yaml + .sops.yaml)
  [ ] Add Gateway API HTTPRoute if external access is required
  [ ] Add Helm values.yaml if using Helm source
  [ ] Extend CNP with app-specific egress rules (database, external APIs, etc.)
```

## What This Skill Does NOT Generate

The following are left for conversational handling after scaffolding:

- SOPS-encrypted secrets and ksops-generator configuration
- Capability-specific CiliumNetworkPolicy rules beyond the baseline
- Helm values tuning or valuesObject content
- Gateway API HTTPRoute or other routing resources
- PNI capability opt-in labels beyond the default `restricted` profile
- StatefulSet, Deployment, Service, or other workload manifests

## Hard Rules

- Do not deploy with `kubectl apply` — all generated files are for GitOps via ArgoCD.
- Use `fromEntities: ["ingress"]` for Gateway API ingress, never `fromEntities: ["world"]`.
- Use port `6443` for kube-apiserver egress (post-DNAT), never `443`.
- Include all six Kubernetes recommended labels on every resource.
- Include PNI labels on the namespace (`platform.io/network-interface-version: v1`, `platform.io/network-profile: restricted`).
- Include PSA labels on the namespace (`enforce: baseline`, `audit: restricted`, `warn: restricted`).
- Always write output in English.
